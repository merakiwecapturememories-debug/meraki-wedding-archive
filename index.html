Scan connected drive

Plug in the HDD, then click "Choose folder" and select the drive's root folder — the top-level one that contains all your event folders side by side (e.g. "01 Chanchal & Anupriya", "02 Priya & Rahul"...). Don't select an individual event's folder — its Photo/Video subfolders would get mistaken for separate events. Each top-level folder you select becomes one event — its name becomes the client name and its size is measured automatically. Works in Chrome or Edge on desktop only.
Note: sizes here use standard decimal GB/TB (same as a drive's rated capacity, e.g. "5TB" on the box). Windows' own Properties dialog shows binary units but mislabels them the same way — so the numbers there will look ~7% smaller than here for the same data. That's a Windows quirk, not an error in either place.

Drive has thousands of files? The browser scan can take a while — use the faster method instead
Browser scanning reads every single file individually, which slows down a lot on drives with tens of thousands of RAW photos/videos. For a near-instant scan instead:
  1. Download scan_drive.py once (needs Python — get it here if you don't have it)
  2. Double-click it, paste the drive's folder path when asked
  3. It finishes in seconds and prints a CSV block — copy it
  4. Paste it below and click "Parse pasted results"
